C 語言控制流程 (Control Flow)

一份給初學者與專業開發者的深度學習指南

條件判斷：if, if-else, if-else ladder

1 基本 if 陳述式

`if` 陳述式用於執行條件性的程式碼。它會評估括號 `()` 中的測試運算式，如果運算式為真 (非零值)，則執行 `if` 區塊內的程式碼。

// 語法
if (測試運��式) {
    // 如果運算式為真，則執行此處的程式碼
}

深度解析： 在 C 語言中，「真」不僅僅是 `true`。任何非零的數值 (如 1, -5, 100) 在條件判斷中都會被視為「真」。只有 `0` 會被視為「假」。這就是為什麼 `if (pointer)` 可以用來檢查指標是否為 `NULL` (因為 `NULL` 的值就是 0)。

2 if-else 陳述式

`if` 陳述式可以有一個選擇性的 `else` 區塊。當 `if` 的測試運算式為假 (值為 0) 時，就會執行 `else` 區塊內的程式碼。

// 範例：檢查一個整數是奇數還是偶數
#include <stdio.h>

int main() {
    int number;
    printf("請輸入一個整數: ");
    scanf("%d", &number);

    // 如果數字除以 2 的餘數為 0，則為偶數
    if (number % 2 == 0) {
        printf("%d 是一個偶數。\n", number);
    } else {
        printf("%d 是一個奇數。\n", number);
    }

    return 0;
}

3 if-else if-else 階梯 (Ladder)

當我們需要從多個可能性中做出選擇時，可以使用 `if-else if-else` 結構。它會依序檢查每個條件，一旦找到為真的條件，就執行對應的程式碼區塊，然後跳過其餘的檢查。

// 範例：比較兩個整數的關係
if (number1 == number2) {
    printf("結果: %d = %d", number1, number2);
} else if (number1 > number2) {
    printf("結果: %d > %d", number1, number2);
} else {
    printf("結果: %d < %d", number1, number2);
}

4 巢狀 if-else (Nested if-else)

我們可以在一個 `if` 或 `else` 區塊中，再放入另一個 `if-else` 陳述式，形成巢狀結構。這有助於處理更複雜的邏輯判斷。

最佳實踐： 過深的巢狀 `if-else` (超過 2-3 層) 會讓程式碼難以閱讀和維護。如果遇到這種情況，可以考慮使用函式、`switch` 陳述式或重新設計邏輯來簡化結構。

初學者注意： 如果 `if` 或 `else` 的主體只有一行程式碼，可以省略大括號 `{}`。但為了程式碼的一致性和可讀性，並避免未來擴充時出錯，強烈建議**永遠**使用大括號。

迴圈結構：for, while, do-while

在程式設計中，迴圈用於重複執行一個程式碼區塊，直到滿足特定條件為止。C 語言提供三種主要的迴圈類型。

1 for 迴圈

`for` 迴圈非常適合在「已知重複次數」的情況下使用。它的語法結構清晰地將迴圈的三個核心要素整合在一起。

// 語法
for (初始化陳述式; 測試運算式; 更新陳述式) {
    // 迴圈主體
}

執行流程：

  1. 初始化：僅在迴圈開始時執行一次。
  2. 測試：在每次迭代前評估測試運算式。若為真，則執行迴圈主體；若為假，則結束迴圈。
  3. 執行：執行迴圈主體內的程式碼。
  4. 更新：執行完迴圈主體後，執行更新陳述式，然後回到第 2 步。
// 範例：計算 1 到 n 的自然數總和
#include <stdio.h>

int main() {
    int num, count, sum = 0;
    printf("請輸入一個正整數: ");
    scanf("%d", &num);

    // for 迴圈在 count 超過 num 時終止
    for (count = 1; count <= num; ++count) {
        sum += count;
    }

    printf("總和 = %d\n", sum);
    return 0;
}

2 while 迴圈

`while` 迴圈是一種「前測式」迴圈，它在每次迭代開始前檢查條件。如果條件為真，則執行迴圈主體。它適用於「不知道確切重複次數，但知道終止條件」的場景。

// 範例：印出 1 到 5
int i = 1;
while (i <= 5) {
    printf("%d\n", i);     ++i; // 如果沒有這行，會造成無窮迴圈！
}

初學者注意： 使用 `while` 迴圈時，務必確保在迴圈主體內有更新條件變數的程式碼 (如 `++i`)，否則測試運算式將永遠為真，導致「無窮迴圈」(Infinite Loop)，程式會卡住直到被強制終止。

3 do-while 迴圈

`do-while` 迴圈是一種「後測式」迴圈。它與 `while` 迴圈非常相似，但有一個關鍵區別：`do-while` 迴圈的**主體至少會被執行一次**，然後才檢查測試運算式。

// 語法
do {
    // 迴圈主體 (至少執行一次)
} while (測試運算式);

這在需要先執行某些操作（例如：向使用者請求輸入）再判斷是否重複的場景中非常有用。

// 範例：不斷要求使用者輸入數字，直到輸入 0 為止
double number, sum = 0.0;
do {
    printf("請輸入一個數字 (輸入 0 結束): ");
    scanf("%lf", &number);
    sum += number;
} while (number != 0.0);

printf("總和 = %.2lf\n", sum);

迴圈控制：break 與 continue

1 break 陳述式

`break` 陳述式用於立即**終止**並跳出其所在的整個迴圈 ( `for`, `while`, `do-while` ) 或 `switch` 陳述式。

// 範例：最多計算 10 個數字的總和，但如果使用者輸入負數，則提前結束
for (i = 1; i <= 10; ++i) {
    printf("輸入 n%d: ", i);     scanf("%lf", &number);

    // 如果使用者輸入負數，就跳出迴圈
    if (number < 0.0) {
        break;
    }
    sum += number;
}

2 continue 陳述式

`continue` 陳述式用於**跳過**目前這次的迭代，並立即開始**下一次**的迭代。

// 範例：計算總和，但忽略所有負數
for (i = 1; i <= 10; ++i) {
    printf("輸入 n%d: ", i);     scanf("%lf", &number);

    // 如果數字是負數，跳過這次加總，直接進入下一次迴圈
    if (number < 0.0) {         continue;
    }

    sum += number;
}

深度解析： 在 `for` 迴圈中，`continue` 會跳到**更新陳述式** (`++i`)；而在 `while` 或 `do-while` 迴圈中，`continue` 會直接跳到**測試運算式**。這個細微的差別有時會影響迴圈的行為。

其他流程控制：switch 與 goto

1 switch 陳述式

`switch` 陳述式提供了一個比 `if-else if` 階梯更清晰的方式來從多個選項中選擇一個執行路徑。它將一個運算式的值與多個 `case` 標籤進行比較。

// 範例：簡易計算機
switch (operation) {
    case '+':
        printf("%.1lf + %.1lf = %.1lf", n1, n2, n1 + n2);
        break;

    case '-':
        printf("%.1lf - %.1lf = %.1lf", n1, n2, n1 - n2);
        break;

    case '*':
        printf("%.1lf * %.1lf = %.1lf", n1, n2, n1 * n2);
        break;

    case '/':
        printf("%.1lf / %.1lf = %.1lf", n1, n2, n1 / n2);
        break;

    default:
        printf("錯誤！運算子不正確。");
}

初學者注意 (Fall-through 行為)： `switch` 的一個重要特性是，如果 `case` 區塊結尾沒有 `break`，程式會繼續執行**下一個** `case` 的程式碼，直到遇到 `break` 或 `switch` 結束為止。這被稱為 "fall-through"。雖然有時可以巧妙運用，但通常是 bug 的來源。**請務必在每個 `case` 的結尾加上 `break`**，除非你非常確定你需要 fall-through 行為。

2 goto 陳述式

`goto` 陳述式允許程式的控制權無條件地轉移到同一個函式內的指定標籤處。

最佳實踐：強烈建議避免使用 `goto`！

濫用 `goto` 會產生所謂的「義大利麵程式碼」(Spaghetti Code)，程式的執行流程像麵條一樣混亂交錯，極難理解、追蹤和維護。幾乎所有的 `goto` 應用場景都有更好、更結構化的替代方案 (如 `if`, `for`, `while`, `break`, `continue` 或將程式碼重構為函式)。

一個極少數被認為可以接受的 `goto` 用法是：從深層巢狀迴圈中一次性跳出到一個統一的錯誤處理或清理區塊。但即便如此，也通常有更好的寫法。

// 一個可以接受 goto 的罕見範例：從巢狀迴圈中跳出
for (...) {
    for (...) {
        while (...) {
            if (發生嚴重錯誤) {
                goto error_cleanup;
            }
        }
    }
}

error_cleanup:
    // 在這裡執行資源釋放或錯誤回報

(A) 跳過目前這次迭代，進入下一次 (B) 立即終止整個迴圈 (C) 程式發生錯誤 (D) 結束目前的函式

3. 在 `switch` 陳述式中，如果一個 `case` 區塊結尾沒有 `break`，會發生什麼？

(A) 程式會報錯 (B) `switch` 結束 (C) 程式會繼續執行下一個 `case` 的程式碼 (Fall-through) (D) 只有 `default` 區塊會被執行

4. 考慮程式碼 `for (i = 0; i < 5; ++i) { if (i == 2) continue; printf("%d ", i); }`，其輸出結果為何？

(A) 0 1 3 4 (B) 0 1 2 3 4 (C) 0 1 (D) 0 1 2

(A) 應該經常用於簡化程式邏輯 (B) 它是唯一能跳出巢狀迴圈的方法 (C) 應極力避免使用，因其會破壞程式結構 (D) `goto` 和 `if` 的功能完全相同
